# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the Cascavel framework, we appreciate your help in disclosing it responsibly.

1. Contact the maintainer **DevFerreiraG** via [LinkedIn](https://www.linkedin.com/in/DevFerreiraG) or open a private security advisory on [GitHub](https://github.com/glferreira-devsecops/Cascavel/security/advisories).
2. Provide a detailed description, reproduction steps, and severity rating.
3. Please avoid publicly disclosing the issue until we have addressed it.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.x     | ✅        |
| 1.x     | ⚠️ Limited |

## Response

We aim to respond within **48 hours** and patch valid issues within **7 business days**.

## Scope

This security policy covers `cascavel.py` and all plugins in `plugins/`. Third-party tools integrated by the framework (nmap, nuclei, etc.) have their own security policies.
